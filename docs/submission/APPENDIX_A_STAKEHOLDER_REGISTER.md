## A.3 Detailed Stakeholder Profiles

### S01: COMP 237 Students (Primary Users)

| Attribute | Description |
|-----------|-------------|
| **Role** | End users of the tutoring system |
| **Needs** | 24/7 learning support, concept clarification, code guidance |
| **Expectations** | Fast responses, accurate information, helpful scaffolding |
| **Concerns** | Academic integrity compliance, privacy of interactions |
| **Success Criteria** | Improved understanding, better grades, reduced anxiety |
| **Communication** | In-app notifications, course announcements |
| **Engagement Level** | High (daily interaction expected) |

### S02: Course Instructor (Content Owner)

| Attribute | Description |
|-----------|-------------|
| **Role** | Academic authority, policy setter, content provider |
| **Needs** | Student learning support, reduced routine question load |
| **Expectations** | System enforces academic integrity, aligns with pedagogy |
| **Concerns** | Students bypassing learning process, misinformation |
| **Success Criteria** | Improved student outcomes, maintained integrity |
| **Communication** | Admin dashboard, email updates |
| **Engagement Level** | Medium (weekly monitoring) |

### S03: Teaching Assistants (Support Staff)

| Attribute | Description |
|-----------|-------------|
| **Role** | Student support, lab instruction |
| **Needs** | Tool to refer students to, reduced repetitive questions |
| **Expectations** | System handles basic questions accurately |
| **Concerns** | Students expecting AI to replace TA help |
| **Success Criteria** | More time for complex student interactions |
| **Communication** | Training sessions, documentation |
| **Engagement Level** | Medium (as-needed reference) |

### S04: Academic Integrity Office (Regulatory)

| Attribute | Description |
|-----------|-------------|
| **Role** | Policy compliance oversight |
| **Needs** | Assurance that AI doesn't enable cheating |
| **Expectations** | Clear policies, audit capability, incident reporting |
| **Concerns** | AI providing assignment solutions |
| **Success Criteria** | No integrity violations from system use |
| **Communication** | Policy documentation, compliance reports |
| **Engagement Level** | Low (audit-driven) |

### S05: IT Department (Technical Support)

| Attribute | Description |
|-----------|-------------|
| **Role** | Infrastructure provisioning, security oversight |
| **Needs** | System stability, security compliance |
| **Expectations** | Standard technologies, minimal support burden |
| **Concerns** | Security vulnerabilities, data breaches |
| **Success Criteria** | Stable operation, no security incidents |
| **Communication** | Technical documentation, incident reports |
| **Engagement Level** | Low (as-needed support) |

### S06: Program Coordinator (Administrative)

| Attribute | Description |
|-----------|-------------|
| **Role** | Academic program oversight |
| **Needs** | Innovation in teaching, positive student outcomes |
| **Expectations** | System enhances program reputation |
| **Concerns** | Scalability to other courses, sustainability |
| **Success Criteria** | Successful pilot, expansion potential |
| **Communication** | Summary reports, presentations |
| **Engagement Level** | Low (milestone reviews) |

### S07: Development Team (Project)

| Attribute | Description |
|-----------|-------------|
| **Role** | System design, implementation, maintenance |
| **Needs** | Clear requirements, stakeholder feedback |
| **Expectations** | Support for technical decisions |
| **Concerns** | Scope creep, unrealistic timelines |
| **Success Criteria** | Functional system, positive user feedback |
| **Communication** | Agile ceremonies, documentation |
| **Engagement Level** | High (continuous development) |
